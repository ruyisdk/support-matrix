# Yocto Milk-V Duo Test Report

## Test Environment

### Operating System Information
- Build System Version: Debian 12 bookworm
- Source Code Link: https://github.com/kinsamanka/meta-milkv
- Reference Installation Document: https://github.com/kinsamanka/meta-milkv/blob/master/README.md

### Hardware Information

- Milk-V Duo 64M
- A USB Power Adapter
- A USB-A to C or USB C to C Cable
- A microSD Card
- A USB to UART Debugger (e.g., CH340, CH341, FT2232, etc.)
- Three Dupont Lines
- Headers pre-soldered on the Milk-V Duo for debugging purposes

## Installation Steps

### Preparing the System Environment

On Debian or Ubuntu:

```shell
sudo apt install gawk wget git diffstat unzip texinfo gcc build-essential chrpath socat cpio python3 python3-pip python3-pexpect xz-utils debianutils iputils-ping python3-git python3-jinja2 libegl1-mesa libsdl1.2-dev python3-subunit mesa-common-dev zstd liblz4-tool file locales libacl1
sudo locale-gen en_US.UTF-8 # skip if locale already set
sudo apt install kas
```

### Download Source Code and Build

Set the board image to be built:

```shell
export KAS_MACHINE=milkv-duo
```

Fetch source code:

```shell
mkdir yocto
cd yocto
git clone https://github.com/kinsamanka/meta-milkv
```

Then build with `kas`:

```shell
kas build meta-milkv/kas-project.yml
```

The resulting system image would be at `build/tmp-musl/deploy/images/milkv-duo/core-image-minimal-milkv-duo.rootfs.wic.gz`.

### Flashing the Image to SD Card

```shell
zcat build/tmp-musl/deploy/images/milkv-duo/core-image-minimal-milkv-duo.rootfs.wic.gz | \
  sudo dd of=/dev/your/device bs=4M iflag=fullblock oflag=direct conv=fsync status=progress
```

### Logging into the System

Logging into the system via the serial port.

Default username: `root`
Default password: none
